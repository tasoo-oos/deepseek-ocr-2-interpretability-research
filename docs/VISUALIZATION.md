# Visualization Workflow

This document describes the visualization code that exists in the current repo.
The active implementation lives under `src/visualization/`, and the main CLI is
`scripts/extract_attention.py`.

## Scope

The visualization stack is focused on the D2E Visual Causal Flow mechanism:

- extract attention weights from the Qwen2 Decoder-as-Encoder
- compare them with the expected mixed causal mask
- map query attention back onto image space
- summarize head specialization and layer evolution

## Current Files

### `src/visualization/attention_viz.py`

Provides `AttentionVisualizer`, which supports:

- `plot_attention_mask()`
- `plot_layer_evolution()`
- `plot_query_to_image()`
- `plot_image_self_attention()`
- `plot_causal_flow()`
- `plot_head_comparison()`
- `plot_entropy_analysis()`
- `create_summary_report()`

### `src/visualization/feature_viz.py`

Provides `FeatureVisualizer` for:

- SAM feature grids
- D2E hidden-state views
- projector-output inspection
- activation-trajectory plots

### `src/visualization/utils.py`

Shared helpers for:

- attention entropy
- region extraction
- expected mask visualization
- flat-to-spatial reshaping
- image overlays

## Extraction Path

The visualization workflow uses:

- `src/models/qwen2_d2e.py`
- `src/analysis/attention_analysis.py`
- `scripts/extract_attention.py`

Minimal example:

```python
import torch
from src.models.qwen2_d2e import build_qwen2_decoder_as_encoder
from src.visualization.attention_viz import AttentionVisualizer

d2e = build_qwen2_decoder_as_encoder(output_attentions=True).eval()
sam_features = torch.randn(1, 896, 16, 16)

with torch.no_grad():
    query_out, attentions, hidden_states, token_type_ids = d2e(
        sam_features,
        output_attentions=True,
    )

viz = AttentionVisualizer(
    attention_weights=[a.cpu() for a in attentions],
    token_type_ids=token_type_ids.cpu(),
    spatial_size=16,
)
fig = viz.plot_attention_mask(layer=0)
```

## Visual Causal Flow Pattern

The expected attention structure is:

```text
          image keys         query keys
image rows   full               blocked
query rows   full               causal
```

More explicitly:

- image -> image: bidirectional
- image -> query: blocked
- query -> image: full cross-attention
- query -> query: lower-triangular causal attention

The expected mask shown by `plot_attention_mask()` is generated by
`create_attention_mask_visualization()` in `src/visualization/utils.py`.

## CLI

Use `scripts/extract_attention.py` for both synthetic and real-model runs.

### Synthetic mode

```bash
uv run python scripts/extract_attention.py \
  --synthetic \
  --output_dir output/attention_viz
```

### Real image mode

```bash
uv run python scripts/extract_attention.py \
  --image_path input/document.jpg \
  --output_dir output/attention_viz \
  --layers 0,6,12,18,23 \
  --full_report
```

## Notes

- `AttentionVisualizer` assumes the number of query tokens equals the number of
  image tokens for the chosen resolution.
- The visualizer operates on extracted attention tensors; it does not own model
  execution itself.
- Real-image extraction uses the current model wrapper and pipeline code, not
  an older `visualization/...` package layout.
